iter.js brings in iterative capabilities over objects by providing an iterator object that can iterate through the enumerable properties of objects.

The project is hosted on GitHub, and the annotated source code is available, an example application, and a list of tutorials iter.js is available for use under the MIT software license.

You can report bugs and discuss features on the GitHub issues page.

Getting Started

Introduction

The current version provides a static iterator that produces a reference object to the object to be iterated over. This allows for the the following: 

for(var i = iter({a:0,b:1,c:2}); i.me != iter.nullptr; ++i.me)
	console.log(i.$);
/* === output ===
0
1
2
*/

The interesting bit is that since i only stores a reference to the object and its property names, the amount of memory i uses is smaller than that of the object.

Loading iter.js

When iter.js loads, it attaches the iter function to whatever scope the file is loaded in. 

function(_) {...}.call(this, this);

This allows for iter.js to attach itself to any library with ease as well as restricting the libraries that iter.js uses. If loading using ajax then using eval to run JS libraries, you can run eval in a given scope and iter will attach to it. 

var thirdParty = {}, iterjsCode = ...;
function (code) { eval(code); }.call(thirdParty, iterjsCode);

If not using ajax to load in, then copy the iter.js file into a function like the following and then presto! iter will attach. 

var thirdParty = {};
function(){/*iter.js code*/}.call(thirdParty);

iter 

iter(object);

The iter function takes in a single argument and creates a static iterator that will iterate through the argument's enumerable properties.

iter Properties

.VERSION => String
.nullptr => nullptr Object
.length => Number

iter.VERSION

Returns the current version of iter.js.

iter.nullptr

The iter.nullptr is an empty object that has been frozen. The valueOf for the iter.nullptr returns NaN and since iterator's .me returns itself or iter.nullptr depending on if it is out of bounds, another way to check if an iterator is within bounds is by using isNaN. 

for(var i = iter([0,1,2,3]); !isNaN(+i.me); ++i.me)
	console.log(i.$);
/* === output ===
0
1
2
3
*/

iter.length 

iter.length([Object]);

Calculates the length of an iterator object or the number of enumerable properties that the object has passed in. 

var o = {a:0, b:1, c:2};
console.log(iter.length(o)); // => 3

Using it on a static iterator will return the length of the object as well as attach a length property to the static_iterator. 

var o = {a:0, b:1, c:2},
    i = iter(o);
console.log(i.length);       // => undefined
console.log(iter.length(i)); // => 3
console.log(i.length);       // => 3

If the function is used on undefined, null, or nullptr the function will return zero automatically. 

console.log(iter.length());             // => 0
console.log(iter.length(undefined));    // => 0
console.log(iter.length(null));         // => 0
console.log(iter.length(iter.nullptr)); // => 0

The reasoning behind not having the length already attached to the static_iterator already is because other objects in the future will not such as the dynamic ones. So, to keep things consistant all calculations for length can be done through iter.length. Also, it allows static_iterator objects to not have unnecessary properties when the length is very uncommon in using iterators in other languages.

Static Iterator 

var i = iter(Object);

A static_iterator is a specialized object that iterates through a given object's enumerable properties. The valueOf for the object returns its current ith position within the object's properties. This combined with the static_iterator's .me property allows for i.me++. The reason for it being called static is because if the object being iterated over gains or loses any properties, the iterator would not know.

Static Iterator Properties

.me => itself
.$ => Object
.scope => Object
.prop => Object
.first => itself
.last => itself
.next => itself
.prev => itself

.me

The most used property of the static_iterator object, .me merely returns the iterator object itself. But, that is only if the current ith position of the iterator object is within the bounds of the object's enumerable properties. If the ith position falls outside of this range, .me returns iter.nullptr. This allows for the detection of whether or not an iterator has reached the end of a loop. 

var i = iter([]);
i.me === iter.nullptr; // => true

This result is because .me is a getter and setter property. When the get is called the property evaluates where the ith position is and determines whether or not to return itself or iter.nullptr. When the set is called the property evaluates whether or not the value attempting to be set is a number. If it is a number (not NaN) then the ith position will be changed to that new value. This allows for the i.me++. 

var i = iter(['a']);
+i == 0 && +i.me == 0; // => true 
i.me++ == 0;           // => true 
+i == 0 && +i.me == 0; // => true

.$

.$ property dereferences the static_iterator and returns the value that it is pointing to. 

var i = iter(["a"]);
console.log(i.$); // => 'a'

This is the same as using i.scope [i.prop].

.scope

.scope returns the obejct that the static_iterator is iterating over. 

var o = {a:0},
	i = iter(o);
o === i.scope; // => true

.prop

.prop returns the property name the static_iterator is currently referencing. If the static_iterator is out of bounds then .prop will return iter.nullptr. 

var i = iter({a:0});
console.log(i.prop); // => "a"
i.me++;
console.log(i.prop); // => iter.nullptr

.first

.first returns the static_iterator and sets the static_iterator to the first enumerable property. 

var i = iter({a:0, b:1, c:2});
console.log(i.prop); // => "a"
i.me+=2;
console.log(i.prop); // => "c"
console.log(i.first.prop); // => "a"

This allows for resetting the static_iterator after a for loop. 

for(var i = iter({a:0,b:1,c:2}); !isNaN(+i.me); ++i.me)
	console.log(i.prop, i.$);
for(i.first; !isNaN(+i.me); ++i.me)
	console.log(i.prop, i.$);
/* === output ===
a : 0
b : 1
c : 2
a : 0
b : 1
c : 2
*/

.last

.last returns the static_iterator and sets the static_iterator to the last enumerable property. 

var i = iter({a:0, b:1, c:2}).last;
console.log(i.prop); // => "c"

This allows for reverse for loops. 

for(var i = iter({a:0, b:1, c:2}).last; !isNaN(+i.me); --i.me)
	console.log(i.prop, ":", i.$);
/* === output ===
c : 2
b : 1
a : 0
*/

.next

.next property will increment the ith value stored within the static_iterator then return the instance. Useful for while loops. 

var o = {a:0,b:1,c:2}, i = iter(o);
do {
    console.log(i.prop, ":", i.$);
} while(!isNaN(+i.next.me));
/* === output ===
a : 0
b : 1
c : 2
*/

Can combine with .prev to create a unique way of looping through a for loop. Since a for loop will run the condition first before running, by first going back one the for loop will increment before checking out of bounds to find it is in bounds. 

var o = {a:0,b:1,c:2};
for(var i = iter(o).prev; !isNaN(+i.next.me);)
    console.log(i.prop, ":", i.$);
/* === output ===
a : 0
b : 1
c : 2
*/

.prev

.prev property will decrement the ith value stored within the static_iterator then return the instance. Useful for while loops. 

var o = {a:0,b:1,c:2}, i = iter(o).last;
do {
    console.log(i.prop, ":", i.$);
} while(!isNaN(+i.prev.me));
/* === output ===
c : 2
b : 1
a : 0
*/

Can combine with .next to create a unique way of looping through a for loop. Since a for loop will run the condition first before running, by first going up one the for loop will decrement before checking out of bounds to find it is in bounds. 

var o = {a:0,b:1,c:2};
for(var i = iter(o).last.next; !isNaN(+i.prev.me);)
    console.log(i.prop, ":", i.$);
/* === output ===
c : 2
b : 1
a : 0
*/
